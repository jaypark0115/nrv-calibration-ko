1. FX10 외부 QSPI 플래시 쓰기 실패

FX10 외부 QSPI 플래시 복구: 잘못된 bitstream 뒤 정상 기록이 실패한 문제

잘못된 bitstream을 외부 QSPI 플래시에 기록한 뒤, 정상 bitstream을 다시 넣으려 해도 기록이 실패했습니다. 프로그램은 erase가 끝났다고 표시했지만, 새 데이터를 쓴 뒤 세 번째 page를 다시 읽어 비교하는 과정에서 실패했습니다. 이 글은 실패 위치를 추적해 플래시를 복구하고 정상 firmware와 bitstream을 다시 기록한 과정입니다.

증상 잘못된 bitstream 뒤 정상 bitstream도 page index 2에서 readback 실패
환경 FX10 bootloader, EZ-USB FX Control Center, 외부 QSPI NOR, Ti180 bitstream
원인 현재 flash mode와 맞지 않는 erase 처리로 앞부분에 이전 데이터가 남음
해결 erase-only helper로 flash를 지우고 0xFF를 확인한 뒤 정상 기록 절차 재실행
결과 erase 검증 PASS, 이후 bitstream 기록 정상 완료

1. 문제 상황

Efinity에서 만든 잘못된 bitstream을 외부 QSPI 플래시에 한 번 기록한 뒤 문제가 시작됐습니다. 이후 정상 bitstream을 다시 넣으려고 해도 프로그램은 erase가 완료됐다고 표시한 뒤 page index 2에서 기록을 멈췄습니다.

화면에는 읽기 비교 단계의 오류가 표시됐습니다. 이후 플래시 앞부분을 확인하니, erase 완료 메시지와 달리 이전 bitstream 데이터가 일부 남아 있었습니다. 남아 있던 값 때문에 새 bitstream을 기록한 뒤 읽은 데이터가 원본과 달라졌습니다.

2. 오류 로그에서 확인한 내용

Total number of sectors to erase : 82
Successfully erased 82 sectors
Total number of bytes to be written : 5343105
Total number of pages to write : 2609
Read buffer mismatch for page index : 2
Failed to write into page index : 2
Firmware download failed

기록 page는 2 KiB였고 index는 0부터 시작했습니다. 따라서 세 번째 page인 page index 2는 0x1000에서 시작합니다. 오류가 이 위치에서 반복된다는 점을 근거로, 기록 파일보다 플래시 앞부분이 실제로 지워졌는지 먼저 확인했습니다.

3. 0x1000에서 실패한 이유: sector 모드

처음에는 host와 센서 레지스터도 확인했습니다. 하지만 장치 연결과 제어는 정상적이었고, 외부 flash ID도 응답했습니다. 문제 범위를 외부 QSPI 플래시의 기록과 erase 과정으로 좁힌 이유입니다.

근본 원인

flash는 hybrid mode였지만, 기존 프로그램은 uniform sector 기준으로 지우고 있었습니다.

QSPI NOR flash에는 앞부분을 작은 4 KiB sector로 나누는 hybrid mode와, 전체를 64 KiB sector로 보는 uniform mode가 있습니다. 이 mode는 Configuration Register 3의 CR3NV[3] bit가 결정합니다. 0이면 hybrid mode, 1이면 uniform mode입니다.

잘못된 bitstream을 기록한 뒤 flash map을 읽어 보니 이 보드는 hybrid mode로 동작하고 있었습니다. 그러나 기존 프로그램은 uniform mode의 지우기 단위를 전제로 erase를 수행하고 있었습니다. 그래서 Successfully erased 82 sectors가 표시돼도 0x1000이 속한 4 KiB 구간에는 이전 데이터가 남았습니다.

NOR flash는 erase가 끝나면 모든 bit가 1, 즉 0xFF가 됩니다. 새 데이터를 기록하기 전에 이전 값이 0xFF로 돌아가야 합니다. 이전 값이 남아 있으면 새 데이터를 다시 읽었을 때 원본과 달라지고, 이번처럼 read buffer mismatch가 발생합니다.

4. 해결: flash를 직접 지운 뒤 다시 기록

기존 프로그램으로 erase를 반복하지 않고, 현재 flash 상태를 직접 처리하는 erase-only helper를 사용했습니다. helper는 외부 flash 전체에 bulk erase 명령 0xC7을 보낸 뒤, 문제가 난 0x1000을 포함한 주소를 읽어 모두 0xFF인지 확인했습니다. 이 과정으로 앞부분에 남아 있던 데이터를 제거했습니다.

확인이 끝난 뒤에는 정상 firmware를 다시 올리고, 마지막으로 정상 bitstream을 기록했습니다. 이 순서로 복구한 뒤에는 같은 위치에서 기록이 멈추지 않았습니다.

5. 해결 검증

helper 로그에서 0x1000을 포함한 주소가 모두 통과한 것을 확인한 뒤, 정상 firmware와 정상 bitstream을 다시 기록했습니다. 이후 FX Control Center에서 firmware download와 bitstream 기록이 정상적으로 완료됐습니다.

확인 항목 수정 전 수정 후
외부 flash ID 응답함 응답함
0x1000 erase 검증 불완전한 상태로 판단 0xFF 확인
복구 순서 erase 완료 표시 후 바로 기록 0xFF 확인 → 정상 firmware → bitstream 재기록
bitstream 기록 Firmware download failed 정상 bitstream 재기록

6. 정리

  • 이번 오류는 정상 bitstream 파일의 문제가 아니라, 외부 QSPI 플래시 앞부분에 이전 데이터가 남아 발생했습니다.
  • page index 2를 주소로 바꾸면 0x1000이었고, 이 주소가 원인을 좁히는 단서가 됐습니다.
  • 복구할 때는 erase 완료 메시지만 보지 않고, 실제 값이 0xFF인지 확인한 뒤 정상 firmware와 bitstream을 기록했습니다.
