SoC/FPGA Bus 학습 노트

이 노트는 숭실대학교 차세대반도체학과 동계 학부생연구인턴에서 진행한 APB, AHB, AXI, AXI-to-APB Bridge 실습을 위키형으로 정리한 글입니다. bus가 실제 회로에서 무엇을 의미하는지부터 각 protocol의 신호, transfer 순서, Lab RTL 구현, SimVision 파형, 최종 프로젝트 로그까지 한 흐름으로 설명합니다.

읽는 순서 APB에서 register access를 먼저 잡고, AHB에서 phase와 wait-state를 본 뒤, AXI의 channel handshake를 이해하고, 마지막으로 AXI-to-APB Bridge에서 protocol 변환을 구현한 흐름입니다.

RTL을 읽는 기본 기준

이 글에서 말하는 RTL은 “하드웨어가 매 clock마다 어떤 값을 저장하고, 어떤 조합논리로 다음 신호를 만들지”를 Verilog로 적은 설계입니다. C 코드처럼 위에서 아래로 한 줄씩 실행되는 프로그램이 아니라, 합성 후 register, mux, comparator, FSM, 배선으로 바뀌는 회로 설명입니다. 그래서 bus RTL을 읽을 때는 신호 이름보다 먼저 “어떤 신호가 register에 저장되는가”, “어떤 조건에서 handshake가 성립하는가”, “주소가 어떤 register나 memory로 decode되는가”를 봐야 합니다.

조합논리

assign이나 always @(*)로 만든 즉시 계산 회로입니다. 주소 비교, mux, enable 생성이 여기에 해당합니다.

순차논리

always @(posedge clk)에서 clock마다 값을 저장하는 회로입니다. address latch, data register, state register가 여기에 해당합니다.

Handshake

보내는 쪽과 받는 쪽이 같은 cycle에 동의해야 transfer가 완료되는 규칙입니다. AXI의 VALID/READY, AHB/APB의 READY가 핵심입니다.

FSM

복잡한 protocol을 상태별로 쪼갠 구조입니다. Bridge에서는 AXI address/data/response와 APB setup/enable을 state로 맞췄습니다.

수업자료 기준 Lab 진행 흐름

전체 과정은 APB register 실습에서 시작해 SRAM과 interrupt로 확장되고, AHB의 phase 구조, AXI의 channel 구조, 최종 AXI-to-APB protocol conversion으로 이어진다. 각 페이지는 단순 활동 기록이 아니라 “개념 정의 → 신호 설명 → RTL 구현 → 검증” 순서로 읽히도록 구성했다.

Lab2~4

APB 기반 register/SRAM/interrupt

주소 decode, write commit, read mux, start command, pending/mask register를 실습했다.

Lab5~6

AHB register와 SP-SRAM

address/data phase, HREADY, zero-wait write와 one-wait read를 확인했다.

Lab7

AXI slave register block

AW/W/B와 AR/R 채널을 분리해 VALID/READY handshake를 검증했다.

Project

AXI-to-APB Bridge

write/read FSM으로 protocol을 변환하고 4-register readback PASS를 확인했다.
